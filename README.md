## Информационно-справочная система парикмахерской

Веб‑приложение на **Spring Boot** для управления работой парикмахерской/салона красоты: учет мастеров, услуг, клиентов и записей, формирование отчетов и базовая аналитика.

- **Назначение**: автоматизировать ежедневную работу салона (расписание, услуги, учет клиентов и выручки).
- **Тип приложения**: монолитное Java Spring Boot приложение с серверной генерацией страниц (Thymeleaf) и REST API.
- **Роль автора**: полный цикл разработки — от проектирования БД и бизнес‑логики до реализации интерфейса и настройки безопасности.

---

## Основной функционал

- **Управление записями клиентов**
  - создание, редактирование и удаление записей;
  - выбор мастера, услуги, даты и времени;
  - проверка доступности слотов.

- **Справочники**
  - **мастера**: ФИО, специализация, контакты, график;
  - **услуги**: название, категория, длительность, стоимость.

- **Пользователи и роли**
  - регистрация и авторизация пользователей;
  - роли (например, администратор, мастер, пользователь);
  - защита URL и операций через Spring Security.

- **Отчеты и статистика**
  - отчеты по записям, услугам, мастерам;
  - базовая статистика (количество записей, выручка и т.п.);
  - экспорт/импорт данных (CSV/JSON).

- **Веб‑интерфейс**
  - страницы входа и регистрации;
  - главная страница с навигацией;
  - формы для работы с записями, мастерами, услугами и отчетами.

## Используемые технологии

- **Back-end**
  - Java 17
  - Spring Boot 3 (Web, Data JPA, Validation)
  - Spring Security 6 (аутентификация, авторизация, шифрование паролей)
  - PostgreSQL + Hibernate

- **Front-end**
  - Thymeleaf (шаблонизатор)
  - HTML5, CSS3, JavaScript

- **Документация и интеграция**
  - Swagger / OpenAPI (`springdoc-openapi-starter-webmvc-ui`)

- **Прочее**
  - Maven
  - Lombok (для сокращения boilerplate-кода)

## Как запустить проект локально

### 1. Предварительные требования

- установлен **Java 17+**;
- установлен **Maven**;
- установлен и запущен **PostgreSQL**.

### 2. Настройка базы данных

1. Создайте базу данных PostgreSQL:
   - имя, указанное по умолчанию в проекте: **`parik_db`**;
   - можно изменить настройки в файле `src/main/resources/application.properties`.
2. Убедитесь, что логин/пароль пользователя БД совпадают с настройками в `application.properties`.

Скрипты создания схемы и начального наполнения находятся в `src/main/resources/db/schema.sql` и `data.sql`.

### 3. Сборка и запуск

```bash
mvn clean package
mvn spring-boot:run
```

Приложение по умолчанию будет доступно по адресу: `http://localhost:8080`.

### 4. Доступ к Swagger UI

После запуска API‑документация будет доступна по URL:

- `http://localhost:8080/swagger-ui.html`

## Структура проекта (кратко)

- `pom.xml` — зависимости и конфигурация Maven/Spring Boot.
- `src/main/java/com/parik/`  
  - `model` — сущности доменной модели (мастер, услуга, запись, пользователь, отчет).
  - `repository` — репозитории Spring Data JPA.
  - `service` — слой бизнес‑логики.
  - `controller` — веб‑контроллеры и REST‑эндпоинты.
  - `config` — конфигурация безопасности и Swagger.
- `src/main/resources/`  
  - `application.properties` — настройки приложения и подключения к БД.
  - `db/schema.sql`, `db/data.sql` — создание таблиц и начальные данные.
  - `templates/` — HTML‑шаблоны (Thymeleaf).
  - `static/` — статические ресурсы (CSS, JS).
